Primeiros computadores

1°Geração: Válvulas

Os computadores que utilizamos no nosso cotidiano passaram por diversas mudanças para serem o que conhecemos hoje. Eles já foram de válvulas, de tamanhos imensos e com pouca capacidade de armazenamento.

Para entendermos como foi a evolução dos computadores, devemos voltar vários atrás, em 1944, durante a segunda guerra mundial. Porém, a idéia de uma máquina que possa realizar cálculos matemáticos é bem mais antiga.

O "primeiro computador" foi o Harvard Mark 1, projetada por um estudante da Harvard, na qual a máquina era capaz de realizar grandes cálculos e foi produzida utilizando válvulas.

O segundo computador foi o Colossus, que foi produzido em 1946 na Alemanha. No entanto, nesse mesmo ano, foi construído o Eniac pelo Estados Unidos.

Foi nessa época que se deu origem ao termo bug

2°Geração: Transistores (1959 a 1965)

Conforme a tecnologia foi avançando, os computadores foram melhorados. Durante essa evolução, os computadores ficaram um pouco mais baratos, tiveram seus tamanhos reduzidos e saíram das válculas para os transistores.

3°geração: Circuitos integrado (1965 a 1970)

Em 1965, outra mudança nos computadores ocorreu, e, dessa vez, foi a implementaçao dos circuitos integrado.

Com a chegada dessa nova tecnologia, os computadores ficaram muito mais rápidos, baratos e, novamente, tiveram seus tamanhos reduzidos

4°Geração: Microprocessadores (a partir de 1971)

A partir de 1971, foi implementado os microprocessadores. Com sua chegada, os computadores passaram a ter seus tamanha diminuídos de novo, ficaram muito mais velozes, os preços foram reduzidos e passaram a ser acessível a todas as pessoas.

foi a partir desse momento que os famosos PCs (computadores pessoais) foram criados. Além disso, também marcou a chegada dos Sistemas Operacionais, como, por exemplo, o Windows

5°Geração: Inteligência Artificial (Dias atuais)

Por fim, chegamos nos dias de hoje, onde temos as inteligências artificiais. As I.A. são capazes de "pensarem por si próprias" e auxiliar os seres humanos em diversas tarefas, além disso, também estão em constante aprendizagem. Ou seja, As I.A. estão sempre aprendendo coisas novas.

Como surgiu a Internet

A internet se deu origem conforme a necessidade de comunicação do ser humano foi aumentado. Contudo, antes da internet ser criada, passaram muitos anos com muitas tecnologias criadas para suprirem essa necessidade.

Aqui está uma pequena linha do tempo da criação dos sistemas de comunicação:

  • Telégrafo (1958) - envia mensagens por código morse;
  • Eniac(1946) - computador criado pelos EUA para cálcular rotas de mísseis;
  • Joseph Carl - pai da computação em nuvem - idealizou uma rede onde todos poderiam se comunicar;
  • Arpanet - realizou a primeira conexão em 1969. A conexão foi entre a universidade da california e o Instituto de pesquisa de Stanford;
  • TCP/IP - modelo criado por Vinton cerf e Robert Kahn. A partir da criação desse modelo, várias microredes foram criadas.
  • WWW - criada por Tim Bernes-Lee, tem o propósito de fazer uma rede na qual todos possam se comunicar sem a necessidade de criar micro redes específicas.

Cliente no linguajar web

Cliente-servidor

O que é o cliente?

O cliente (ou usuário) é quem irá requisitar serviços e consumi-los, sendo uma das pessoas mais importante para o desenvolvedor.

O que é o servidor?

O servidor é um computador mais robusto que irá atender aos pedidos dos clientes

O que é uma rede cliente-servidor?

De forma resumida, uma rede cliente-servidor consiste em uma conexão entre computadores usuários (os clientes) e um servidor (um computador mais poderoso capaz de atender às necessidades dos clientes). Nesta rede, os clientes poderão se conectar à internet, utilizando-se navegadores web para se comunicar com o servidor e utilizar a internet.

Os navegadores gravam na memória cache da máquina do cliente as informações recente a qual ele utilizou. Dessa maneira, não é necessário requisitar diversas vezes o mesmo serviço ao servidor, aumentando a velocidade de resposta.

Navegadores Web

Os navegadores são programas gratuitos que interpretam os arquivos do servidor web. Além disso, eles seguem padrões W3C, no entanto, possuem pequenas diferenças de interpretação entre browsers.

  • Chrome;

Aplicação Web

Muita gente não sabe a diferença entre aplicação web e aplicativo. Por mais que possam parecer o mesmo, existe diferenças entre si.

Diferença entre aplicação web e aplicativo

Uma aplicação web é utilizada através dos navegadores (chrome, microsoft edge etc.), sem ser necessário a instalação dela. Por exemplo: utilizar o youtube através do google chrome.

Já quando vc instala o youtube, ou qualquer outro app, ele deixa de ser uma aplicação web, pois a aplicação é aberta através do dispositivo usuário, e não pelo browser.

Dispositivos móveis

Nos dias de hoje, é bem comum olhar em sua volta e ver que praticamente todas as pessoas estão usando algum dispositivo móvel (celular, tablet, smart watch, etc.).

Cerca de 99% dos latinos-americanos utilizam e já baixaram aplicativos em seus dispositivos.

É de suma importância ter em mente que existe muitos dispositivos diferentes ao criar um site ou aplicativo, pois cada um tem sua compatibilidade específica. Por isso é bom citar o uso dos dispositivos móveis no nosso cotidiano.

Servidor

Os servidores são os responsáveis por disponibilizar os serviços que utilizamos não só na internet, mas também em redes locais. Eles possuem processadores mais potentes, armazenamentos maiores e mais velozes, e muito mais.

Os servidores precisam ser mais robustos do que outro dispositivos para poderem atender as várias requisições dos usuários, já que são eles que permitem o cliente utilizar aplicações web, salvar seus dados, entre outros. Além disso seus tamanhos podem variar, sendo que os maiores são chamdos de datacenter

Existem vários servidores com diferentes tipos de responsabilidades, como, por exemplo, um servidor feito para armazenar arquivos dos uuários.

Todo servidor é composto por uma parte física (hardwares) e uma parte lógica (software). Cada uma trabalha em conjunto para conseguir atender às necessidades das pessoas usuárias.

Hardwares

Planos de hospedagem

É possível criar seu próprio servidor, mas isso demanda tempo e recursos. É aí que entra os planos de hospedagens.

Um plano de hospedagem é contratar um servidor pago ou gratuito para conseguir usá-lo para seu site.

Tipos de servidores

Como dito anteriormente, cada servidor possui sua função. Abaixo estará uma lista desses tipos com suas respectivas funções:

  • Proxy: gerencia o acesso dos usuários a certas informações da internet;
  • Firewall: gerencia o acesso externo ao servidor;
  • Database: banco de dados acessado por outro servidor;
  • Intranet: servidor web local sem internet;
  • DNS: Relaciona nomes com IPs.
Servidores gerais
  • Servidor de arquivos: servidor central para armazenagem de arquivos;
  • Servidor de segurança: firewall;
  • Servidor de streaming: realiza transmmissões de dados (vídeo);
  • Servidor de e-mail:: gerencia, armazena e envia e-mails transmitidos;
  • Servidor Web: Disponibiliza arquivos HTML, Javascript, CSS, entre outros.

Hospedagem de sites

Como citado anteriormente, ao criar um site, podemos enviá-lo para um servidor. Fazendo isso, outras pessoas pode acessar seu site.

É bom dizer sobre o certificado de conexão segura, chamado certificado SSL. Ele que mostra aos outros usuários que seu site possui uma conexão segura, fazendo com que as pessoas se sintam tranquilas utilizando seu site.

Para identificar se um site possui criptografia, basta verificar na barra de URL se está escrito HTTPS ao invés de HTTP. A diferença entre os dois é que o HTTPS possui criptografia e o HTTP não.

HTML não é linguagem de programação

Por incrível que pareça, HTML não é uma linguaegm de programação. Como no próprio nome diz, Hyper Text Markup Language (Linguagem de Marcação de HiperTexto), é um marcador de texto. Onde ele delimita textos por nomes que o navegador consegue interpretar.

Cada delimitador tem uma função e comportamento específico.

Exemplo de delimitador em HTML

O HTML tem seus delimitadores de textos com suas funções específicas. Os delimitadores são chamados de tags. Aqui será mostrado algumas delas.

Imagine que cada delimitador esteja dentro de <>.

  • p: inicia um paragráfo.
  • /p: fecha o parágrafo.
  • strong: deixa o texto em negrito.
  • /strong: para de deixar as palavras em negrito.
  • i: deixa o texto em itálico.
  • /i: para de deixar o texto e itálico.

Exemplo do uso dessas tags:

< p >

Estou aprendendo sobre < strong >< i >tags < /i >< /strong >!

< /p >

Resultado:

Estou aprendendo sobre tags!

Estrutura básica do HTML

Antes nos aprofundarmos mais sobre HTML, temo que entender certas coisas.

  • Tags: Consiste em <> com um nome dentro. Existe duas versões de tag, as que abrem (< nome da tag >) e as que fecham (< /nome da tag >).

O HTML consiste em 3 partes, que são o html, a cabeça (head) e o corpo (body).

  • html: indica que tudo que está entre faz parte do html.
  • head: carrega informações antes de mostrar ao usuário.
  • body: mostra, de fato, as informações ao usuário.

Listas ordenadas e não ordenadas

Lista ordenada

Uma lista ordenada é uma lista com os itens numerados de 1 até X. Ela possui as tags ol (abre a lista ordenada) e a tag li (insere o texto como um item da lista).

Abaixo está um exemplo de uma lista ordenada:

  1. Item de número 1.
  2. Item de número 2.

Lista não ordenada

Uma lista não ordenada é uma lista que não numera seus itens, mas deixa-os marcado. Ela possui as tags ul (abre a lista não ordenada) e li (insere o texto como um item da lista).

Abaixo está um exemplo de uma lista não ordenada:

  • Item 1.
  • Item 2.

Links

Utilizando a tag a podemos criar "links". Os "links" são âncoras que ligam o site com outro site, imagem, ou até outra parte dele mesmo. São os links que redirecionam o usuário de um lado para o outro.

Falando sobre tags

Como dito anteriormente, as tags são os delimitadores do HTML que têm resposabilidades diferentes e consistem em uma tag que abre e uma tag que fecha.

No entanto, existe tags que não precisam fechá-las, como a tag hr por exemplo. Basta colocar uma barra "/" no final da tag.

Lista de tags do HTML

  • html: indica que tudo faz parte do site
  • head: carrega as informações antes de mostrar oa cliente.
  • body: mostra as informações ao usuário.
  • i: deixa em itálico.
  • strong: deixa em negrito.
  • input: cria um campo para o usuário digitar.
  • img: permite inserir uma imagem através de seu link.
  • h1...h6: títulos e sub-títulos.
  • p: inicia um parágrafo.
  • blockquote: faz o texto ser uma citação/observação.
  • u: sublinha o texto.
  • mark: marca o texto.
  • sup: eleva o texto. EX: A -> A
  • hr: insere uma linha
  • small: diminui o tamanho do texto.
  • a: cria uma "âncora" para outro site ou a outra parte da mesma página.
  • ol: lista numeradas.
  • ul: lista não numerada, mas com marcações.
  • li: item da lista.

Atributos das Tags

Os atributos são propriedades das tags que específicam uma determinada função. Esses atributos podem ser gerais (funcionam em qualquer tag) ou específicos da tag (só funciona em certas tags).

Abaixo está uma lista de alguns atributos gerais:

Lista de atributos gerais das tags

  • id: serve para identificar um item.
  • style: serve para colocar comandos diretos do CSS como, por exemplo, mudar a cor de um texto.
  • class: padroniza tudo que estiver com o mesmo nome (id) escolhido.
  • width: muda a largura do texto ou imagem
  • title: define um título ao texto ou imagem.
